# 前端修改如何生效

## 原因

用 **Docker Compose** 时，前端是**构建一次**后由 Nginx 提供静态文件，**没有挂载源码**，所以你在本机改的代码不会自动更新到页面上。

## 做法（二选一）

### 方式一：本地跑前端（推荐，改完保存就能看到）

1. 后端和数据库继续用 Docker（在项目根目录）：
   ```bash
   docker compose up -d db backend
   ```
2. 单独用本地开发服务器跑前端：
   ```bash
   cd frontend
   bun run dev
   ```
3. 浏览器打开：**http://localhost:5173**  
   修改 `frontend/src` 下任意文件并保存，页面会自动刷新，就能看到效果。

### 方式二：改完代码后重新构建 Docker 前端

每次改完前端代码后执行：

```bash
docker compose build frontend
docker compose up -d frontend
```

然后再刷新浏览器。适合不想在本地装 Node/Bun 的情况，但每次改代码都要重新构建。

---

当前已在 **首页 Dashboard**（`src/routes/_layout/index.tsx`）里加了「请输入关键词搜索」的搜索框，用**方式一**跑起来后，登录进首页就能看到。
